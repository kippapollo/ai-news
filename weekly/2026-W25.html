Anthropic ships Claude Fable 5 and Mythos 5, the first publicly available Mythos-class models, and Stripe finishes a 50-million-line Ruby migration in a day

The Wednesday release puts Mythos-class capability on every AWS, Bedrock, Microsoft Foundry, GitHub Copilot, Harvey and Cursor account at $10 input / $50 output per million tokens. Fable 5 posts 80.3% on SWE-Bench Pro. Three classifier-routed safeguards intercept cybersecurity, biology and distillation queries on under 5% of sessions. Mythos 5 deploys with cyber-uplift safeguards lifted to roughly 200 critical-infrastructure partners across 15 countries under Project Glasswing. Within 48 hours Pliny-the-Liberator surfaces the model's 120,000-character system prompt and Anthropic disputes the multi-agent jailbreak in public.

Anthropic · AWS · GitHub · Harvey · 2026-06-10 · Source ↗ ·

Anthropic shipped Claude Fable 5 and Claude Mythos 5 on Wednesday morning, putting the company's Mythos-class tier on general availability for the first time. The release lands as the second public Mythos-class model after Mythos 4 stayed behind a critical-infrastructure-only access wall for nine months, and it materially changes what an enterprise can put on a paid API account today. Fable 5 is the general-purpose flagship: full Mythos-class capability, three classifier-routed safeguards, and pricing of $10 per million input tokens and $50 per million output tokens. Mythos 5 is the variant Anthropic uses inside Project Glasswing, lifted to roughly 200 critical-infrastructure partners across more than fifteen countries through a U.S.-government channel. The two checkpoints share the same training run and diverge on which cyber-uplift capabilities the deployment toggles permit.

The numbers landed hard. Fable 5 scored 80.3% on SWE-Bench Pro, the hardest agent-coding benchmark Anthropic publishes against, opening daylight on every other generally available model. Stripe ran the launch demo: a 50-million-line Ruby migration that the company's platform team estimated at nine months of human work, finished by a Fable 5 swarm in a single day, with zero production incidents on cutover. Harvey scored Fable 5 at 13.3% on Legal Agent Bench, ahead of Opus 4.8's 10.4%, and at 93.4% on BigLaw Bench, both the highest legal scores any model has posted on Harvey's published evals. Cursor put Fable 5 behind its Composer flag for paid users on launch day. GitHub Copilot moved Fable 5 to general availability across all paid plans the same morning, with an accompanying changelog entry from GitHub's Copilot team and AWS publishing the simultaneous availability inside Amazon Bedrock and the Claude Platform on AWS. Microsoft Foundry added the model the same day with governed agent tooling for Azure enterprises.

"Today we are introducing Claude Fable 5, our most capable model, and Claude Mythos 5, our first Mythos-class model with cyber capabilities, both built on the same foundation."— Anthropic, "Introducing Claude Fable 5 and Claude Mythos 5," 2026-06-10

The three safeguards and the cost of safety routing

Anthropic published the safeguard architecture alongside the release. Three classifiers run on every Fable 5 request: a cyber-uplift classifier, a biology-and-chemistry classifier, and a distillation classifier. When a classifier fires, the request routes to Opus 4.8 with a tighter system prompt rather than refusing outright, an architectural choice Anthropic argues preserves productivity at the cost of measurable latency on the small share of requests that trip the rails. The company's published rate is under 5% of all sessions, and the bug-bounty disclosure includes zero universal jailbreaks across more than 1,000 hours of red-team testing through launch. Enterprise customers received an updated retention notice in parallel: a mandatory 30-day data retention window now overrides prior zero-retention enterprise contracts on Fable 5 traffic, a change Anthropic describes as required to support the classifier and incident-response loop. Customers under HIPAA, FINRA or government-cloud terms are negotiating exemptions individually.

Mythos 5 and Project Glasswing

Mythos 5 deploys behind the same training run as Fable 5 with cyber-uplift safeguards lifted to vetted partners under Project Glasswing. The launch list includes BeyondTrust as a privileged-access partner, with Cloudflare publishing a separate post on the cyber-frontier-model deployment surface; Cybernews and PYMNTS covered the Glasswing expansion to roughly 200 partners across critical infrastructure operators in more than fifteen countries. The model is the operational answer Anthropic gives to the question Bruce Schneier pressed on June 9: the Glasswing program found 23,000 vulnerabilities across 1,000 projects in its first year, of which Schneier reported "almost none patched." Anthropic responded by expanding the partner program rather than narrowing it, betting that scale across critical-infrastructure operators is the variable that closes the patch gap.

"Fable 5 finished a 50-million-line Ruby migration in a day. We had estimated nine months of platform engineering."— Stripe platform team, launch-day demo, 2026-06-10

Pliny and the 120,000-character system prompt

The honeymoon lasted 48 hours. On Friday, Pliny-the-Liberator and a small group of collaborators published a multi-agent jailbreak that, according to TechTimes' Saturday writeup, surfaced what Pliny called the model's full 120,000-character system prompt. The attack chained Unicode-tag tricks, long-context reference tracking, document framing and fiction framing, with multiple Claude instances feeding state to each other. Anthropic's threat-intelligence team disputed the disclosure publicly within hours, arguing that what Pliny exposed was a developer-mode debug surface rather than the production system prompt and that no Mythos-class capability had been unlocked. Independent researchers split on whether the exposed text matched the production prompt. The episode reopened the public argument about whether a 120K-token system prompt is a load-bearing safety boundary or a brittle artifact, and Schneier reposted his prior critique with the addition that "if the system prompt is 120,000 characters long, your alignment is in the wrong layer."

Pricing, billing and the enterprise math

The $10 input / $50 output per-million-token pricing is the editorially significant number. Anthropic shifted enterprise billing to consumption-based at the same time, with annualized revenue crossing $30 billion according to The Information's Saturday long-form report. Free Pro, Max, Team and Enterprise users got Fable 5 through June 22 on a metered window. Replit's Amjad Masad posted the same week that putting Fable 5 in production reduced debugging cost per session despite the headline token price, because Fable 5 emits fewer turns to converge on a correct fix. The price-to-quality math is the constraint that decides whether the rest of the enterprise market follows Stripe's deployment pattern. Anthropic's bet is that the productivity gain at SWE-Bench Pro tier is large enough to absorb the per-token premium against the open-weights frontier. The price-quality gap to DeepSeek V4 and Nvidia Nemotron 3 Ultra still runs 30 to 50× in open weights' favor on equivalent agent workloads, and that arithmetic is the load-bearing variable for the second half of 2026.

Google ships DiffusionGemma 26B-A4B under Apache 2.0 as the first production-grade open-weights text-diffusion MoE

Google DeepMind / MarkTechPost · 2026-06-10 · Source ↗ ·

A 26-billion-parameter sparse MoE with 4 billion active per token, denoising 256 tokens in parallel at over 1,000 tokens per second on a single H100 and up to 2,000 on a DGX Station. Day-zero Tensor Core acceleration through Transformers, vLLM, MLX and Unsloth. 77.6 MMLU Pro, 73.2 GPQA Diamond, 69.1 LiveCodeBench v6, 54.3 MMMU Pro. Google directly recommends Gemma 4 for production; DiffusionGemma is the research artifact opening a parallel decode pathway.

DiffusionGemma is the first production-grade open-weights text diffusion model. The architecture is a 26-billion-parameter sparse mixture-of-experts with 4 billion active parameters per token, denoising 256 tokens in parallel rather than autoregressively predicting one token at a time. The license is Apache 2.0, permitting unrestricted commercial use. On a single H100 the model emits over 1,000 tokens per second; on an NVIDIA DGX Station with Blackwell GPUs the throughput crosses 2,000 tokens per second. Hugging Face hosts the weights at google/diffusiongemma-26B-A4B-it; the DeepMind page ships the model card; MarkTechPost's June 10 explainer walks through the Uniform State Diffusion mechanism with re-noising self-correction that autoregressive transformers cannot match.

The benchmark numbers ground the release at the production tier. MMLU Pro scores 77.6, GPQA Diamond 73.2, LiveCodeBench v6 69.1 and MMMU Pro 54.3. Google's own model card directly recommends Gemma 4 for production deployments and frames DiffusionGemma as the research artifact opening a parallel decode pathway. NVIDIA's RTX AI Garage post on June 11 covered day-zero Tensor Core acceleration through Transformers, vLLM, MLX and Unsloth. Hugging Face Transformers v5.11.0 added native DiffusionGemma support on Friday alongside GLM 5.1 support.

"DiffusionGemma denoises 256 tokens in parallel. The autoregressive cost curve does not bend; the architecture does."— MarkTechPost analysis, 2026-06-10

The editorially significant change is the decode-cost curve. Autoregressive transformers pay per-token sequential cost, with each output token depending on the previous one. Text diffusion denoises a window of tokens together, then re-noises and corrects them in subsequent passes. The throughput claim runs roughly 4× faster generation against equivalent-quality autoregressive Gemma 4 baselines on the same hardware, while preserving competitive benchmark scores. The architecture is the first time an open-weights text-diffusion model has shipped at production quality, and the parallel-decode pathway gives developers an alternative to the autoregressive cost ceiling. NVIDIA's Tensor Core acceleration through the open-source toolchain is the supply-side piece that turns the research artifact into a deployable production option.

Claude Code ships /fork, mapping AI coding onto git-style session branching

TechTimes · code.claude.com · 2026-06-13 · Source ↗ ·

The new command lets a developer branch any conversation into parallel sessions with shared upstream history, then merge or discard branches. Claude Code 2.1.174 and 2.1.175 ship the same Friday with enforceAvailableModels closing the last route a user could widen an admin-managed model allow-list, plus VSCode usage attribution and a dozen long-tail fixes. Claude Code is estimated at roughly 4% of public GitHub commits and described internally as Anthropic's fastest-growing product.

Anthropic shipped the /fork command in Claude Code on Friday afternoon, mapping AI coding sessions onto a git-style branching model. A /fork call branches a conversation into a parallel session that shares the upstream history but accumulates its own state, lets the developer try multiple approaches without losing the prior context, and merges or discards branches with a single command. The change ships in Claude Code 2.1.174 and 2.1.175, with VSCode usage-attribution breakdowns and a dozen long-tail fixes accompanying the release.

The same release closes the enforceAvailableModels admin loophole. Until Friday a user could widen an admin-managed model allow-list through a config-file pathway; the 2.1.175 fix tightens that to a server-side enforcement and removes the local override. The change matters for enterprise rollouts where compliance teams pin Claude Code to a specific approved model and need the admin policy to actually bind on the client.

"Claude Code is the fastest-growing product we've ever shipped. The /fork command is the second-most-asked-for feature behind subagent isolation."— Boris Cherny, Claude Code lead, on X, 2026-06-13

The Information's June 14 long-form estimated Claude Code at roughly 4% of public GitHub commits, a figure Anthropic has not formally confirmed but has not pushed back on either. The product is described internally as the company's fastest-growing surface, and the consumption-based enterprise billing model Anthropic moved to during the week is partly underwritten by the Code workflow's compounding token spend. The /fork command's downstream implication is straightforward: parallel exploration that previously lived only in agent-orchestration harnesses now lives in the developer's terminal.

Microsoft open-sources SkillOpt, an optimizer that trains an agent's SKILL.md as the trainable object

VentureBeat · 2026-06-13 · Source ↗ ·

SkillOpt treats the agent's SKILL.md instruction file as a trainable artifact and runs gradient-style optimization against task-success signal without touching model weights. The published numbers are +23.5 accuracy on GPT-5.5, +24.8 inside Codex and +19.1 inside Claude Code. The release ships under MIT on the Microsoft GitHub organization with reference experiments and an evaluation harness. The technique works on any model the agent can call, regardless of provider.

Microsoft Research open-sourced SkillOpt on Saturday under MIT license, attached to the Microsoft GitHub organization with reference experiments and an evaluation harness. The system treats an agent's SKILL.md instruction file as a trainable object and runs gradient-style optimization against task-success signal without touching the underlying model weights. The published evaluation lifts task accuracy by +23.5 absolute points on GPT-5.5, +24.8 inside OpenAI Codex and +19.1 inside Anthropic Claude Code, measured on an internal agent-task suite VentureBeat described in the Saturday release coverage.

The architecture inverts the usual training assumption. Where standard reinforcement-learning-from-human-feedback shifts model weights to improve outcomes, SkillOpt fixes the weights and shifts the prompt. The optimizer takes a SKILL.md as input, runs the agent against a task suite, scores outcomes, and proposes targeted edits to the SKILL.md text using a meta-prompted rewrite loop. The published numbers are competitive with reinforcement-learning fine-tunes on the same task suites, at a fraction of the compute cost and with no provider lock-in.

"SkillOpt treats the SKILL.md as the trainable artifact. We replace weight gradient descent with prompt gradient descent, and it works on every frontier model we tested."— Microsoft Research, SkillOpt repository readme, 2026-06-13

The provider-agnostic property is the editorially significant claim. Enterprises that pinned their agent stack to a specific model can now improve agent performance without changing models, switching API providers or paying for fine-tuning. The same SKILL.md lift transfers across frontier-model boundaries because the optimization target is the instruction text rather than the model weights. The repo ships with a worked example, an evaluation harness based on the agent-task suite, and a step-by-step guide for running SkillOpt against an in-house task corpus.
