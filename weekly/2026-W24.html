Microsoft Build 2026 reframes Windows as an agent platform: MAI-Thinking-1, MXC, Project Solara, Discovery and Majorana 2 land in one keynote week

Microsoft used the week to put a complete agent stack on the table. MAI-Thinking-1 hits 97.0% AIME 2025 and 53% SWE-Bench Pro trained on Maia 200 silicon. MXC drops an OS-level execution sandbox with OpenAI and Nvidia as launch partners. Project Solara opens an agent-first Android device OS. GitHub Copilot Workspace reaches GA. Microsoft Discovery ships with Majorana 2 as the flagship reference workload and pulls the quantum timeline to 2029.

Microsoft AI · Thurrott · VentureBeat · 2026-06-02 — 2026-06-06 · Source ↗ ·

Microsoft used Build 2026 to put a full agent stack on the table, then spent the rest of the week shipping the pieces. By Saturday the picture was clear: a flagship reasoning model trained from scratch on Microsoft's own silicon, an OS-level execution sandbox with OpenAI and Nvidia as launch partners, an agent-first device operating system, the general-availability release of GitHub Copilot Workspace, and the production-ready debut of Microsoft Discovery with a Majorana 2 chemistry workload as the flagship reference. The strategic message Satya Nadella opened with on June 2 was that Windows is no longer a desktop operating system that happens to run AI applications. Windows is the agent platform, and every other Microsoft surface is being rebuilt around that primitive.

MAI-Thinking-1 is the headline. Microsoft AI's introduction post and Neowin's June 6 deep dive confirm a 35-billion-active-parameter Mixture-of-Experts model with a 256K context window, trained from scratch on commercially licensed enterprise data with no distillation from OpenAI or any other third-party system. Pre-training and post-training ran on Microsoft's own Maia 200 silicon and was benchmarked head-to-head against an Nvidia GB200 cluster, which Mustafa Suleyman cited on stage as evidence that the Maia line is production-competitive with Blackwell for frontier training. The benchmark claim is sharp: 97.0% on AIME 2025 and 94.5% on AIME 2026, the standard hard-math reasoning evaluations; 53% on SWE-Bench Pro, the August 2025 coding benchmark that has become the industry's hardest agent-coding evaluation, placing MAI-Thinking-1 alongside Claude Opus 4.6 on the most difficult coding tasks the eval ships with. User-preference testing during the staged rollout favored MAI-Thinking-1 over Claude Sonnet 4.6 on agentic coding work. The model is the default in VS Code immediately, ships on Microsoft Foundry, is hosted on OpenRouter, Fireworks and Baseten, and is available through the GitHub Copilot stack on all paid plans.

"Most importantly, MAI-Thinking-1 is at 53% on SWE-Bench Pro, placing it right alongside Opus 4.6 on one of the toughest coding benchmarks."— Microsoft AI, MAI Build keynote, 2026-06-02

MXC and the agent runtime

VentureBeat's June 6 story put the second piece on the record. MXC is an OS-level execution sandbox for AI agents, shipping in Windows with OpenAI, Nvidia, Manus and Nous Research as launch partners. It runs each agent inside a policy-driven process-isolation envelope with explicit declarations for filesystem, network, registry, peripheral and inter-agent permissions, and it composes with the Execution Containers preview Microsoft pushed at Build's developer keynote. Microsoft positioned MXC as the missing primitive under Windows Agent Framework: where the framework gives developers a way to express an agent's tools and goals, MXC gives the operating system a way to enforce what the agent is actually allowed to do at runtime. The early adopter list is the story under the story. OpenAI is endorsing it because Codex Computer Use on Windows ships inside it. Nvidia is endorsing it because the Isaac GR00T runtime and the Nemotron 3 family route their Windows execution through MXC. Manus and Nous Research are endorsing it because the agent ecosystem that has lived in Linux containers needs a Windows answer to migrate enterprise workloads.

Project Solara, Aion, and the agent-first device

Project Solara is the agent-first device OS Microsoft is building on top of Android in partnership with MediaTek and Qualcomm. The reveal at Build on June 2 named two reference designs, an MediaTek Dimensity 9500 phone and a Qualcomm Snapdragon X3 Elite tablet, with shipping partners targeted for early 2027. Solara replaces the Android launcher with an agent surface, treats apps as tools the agent can call rather than icons the user taps, and runs Aion 1.0 Instruct as the on-device small model. Aion 1.0 Plan is the companion 14-billion-parameter on-device planner with tool-calling, intended to run agent loops without leaving the device. The Windows team shipped Aion 1.0 alongside MAI-Thinking-1 the same week, plus WSL 3 with paravirtualized NPU and GPU passthrough through Windows Update, coreutils for Windows in general availability, Surface RTX Spark Dev Box and the DGX Station for Windows, and the Windows Agent Store with an 85/15 developer revenue share.

GitHub Copilot Workspace and the developer surface

GitHub Copilot Workspace reached general availability the same week, with Fleet for multi-agent orchestration, Autopilot for unattended runs, and a new standalone Copilot desktop app on Windows 11, Windows on Arm, Mac and Linux. Tech Times and Visual Studio Magazine reporting put Project Polaris in as the default coding model under the GitHub Copilot hood beginning in August, replacing GPT-4 Turbo as the in-editor engine. MAI-Code-1-Flash is the coding-specific small model in the MAI bundle: optimized for in-editor completion and rapid agent tasks, with a 16-point lead over Claude Haiku 4.5 on SWE-Bench Pro (51.2% versus 35.2%) and up to 60% fewer tokens on SWE-Bench Verified at the same accuracy. Microsoft is rolling MAI-Code-1-Flash to all GitHub Copilot plans on launch. The combined picture is that Microsoft now competes at every model tier with a proprietary substitute for Anthropic and OpenAI dependencies, hedging cost and supply at once.

"With Discovery generally available we can give every researcher the same closed-loop scientific reasoning that the Majorana team used to compress a six-month chemistry program into ten days."— Satya Nadella, Microsoft CEO, Build 2026 day-two keynote, 2026-06-03

Microsoft Discovery, Majorana 2 and the quantum pull-in

Microsoft Discovery hit general availability on June 3 with a desktop app preview for individual researchers and small teams. The product is an evidence-to-hypothesis loop, the same closed-loop scientific reasoning Microsoft used internally to design the Majorana 2 topological-qubit fabrication process. The flagship reference workload Microsoft published the same day described a switch from aluminum to lead as the superconducting material in the qubit substrate, identified by Discovery in a ten-day search across a 10⁷-candidate chemical space rather than the six-month bench program it had previously required. The disclosure pulled Microsoft's quantum timeline forward to 2029. Microsoft Source's June 3 piece framed this as the moment quantum and AI converged on Microsoft's roadmap, with Discovery as the bridge and Majorana 2 as the proof point.

The shape of the bet

Ben Thompson's June 5 Stratechery essay argued that Build 2026 was the week the inference cost curve bent enough for integration giants to reclaim the runtime from frontier labs. Microsoft is the case study. Every layer of the agent stack now has a Microsoft answer: model (MAI-Thinking-1 and the seven-model bundle), runtime (MXC and Execution Containers), device (Project Solara, Aion, Surface RTX Spark), developer surface (GitHub Copilot Workspace, the standalone Copilot app, Polaris) and scientific workload (Discovery, Majorana 2). The OpenAI partnership stays in place, but the dependency at every layer has been hedged. The Maia 200 head-to-head against GB200 is the substantive part: if Microsoft's silicon can train a competitive frontier model, the multi-year Nvidia exposure also gets hedged. None of these moves alone would reorder the stack. Together, in one week, they remove the structural reason an enterprise would pick a non-Microsoft AI partner for production workloads inside the Windows perimeter.

"Windows is the agent platform now. Every other surface inherits that."— Satya Nadella, Build 2026 opening keynote, 2026-06-02

Risks remain. The MAI-Thinking-1 benchmark claims are internal Microsoft numbers; the third-party reproduction window has not opened. SWE-Bench Pro's score variance between harnesses is wide, and 53% on Microsoft's harness may differ from what other operators measure. The MXC sandbox introduces a Windows-only runtime surface that fragments the agent ecosystem just as MCP has been pulling it toward an open protocol. Project Solara is a 2027 ship target on partner hardware Microsoft does not control. Discovery's Majorana 2 reference is one workload, not a generalizable claim about closed-loop scientific reasoning. None of those caveats undoes the week. They define what the next two quarters have to prove.

Nvidia ships Nemotron 3 Ultra at 550B open weights and closes the agentic decode-throughput gap with China

Nvidia Research / MarkTechPost · 2026-06-04 · Source ↗ ·

A 550-billion-parameter hybrid Mamba-Attention MoE with 55B active per token, 1-million-token context, and weights on Hugging Face, OpenRouter, Together AI and Perplexity under OpenMDW-1.1. 71.9 on SWE-Bench Verified, 94.7 on RULER at 1M tokens, 5.9× faster than GLM-5.1 on GB200 under NVFP4 at 8K/64K. The first US open-weights model that closes the agentic decode-throughput gap with the Chinese frontier.

Nvidia released Nemotron 3 Ultra on June 4 as the production-tier member of the Nemotron 3 family the company first previewed at GTC in December 2025. The configuration is a 550-billion-parameter sparse mixture-of-experts with 55 billion active per token, 108 layers, 8,192 model dimension, 512 experts per layer routing the top 22 per token, 64 query heads and 2 key-value heads. The asymmetric attention shrinks the KV cache, which matters when decode dominates an agent's runtime. The architecture is a hybrid Mamba-Attention design rather than a pure Transformer. Mamba's constant per-step decode cost scales linearly with sequence length, while attention's quadratic cost dominates long-running agents that emit tens of thousands of output tokens per task.

The model ships on Hugging Face in both BF16 and NVFP4 checkpoints, on OpenRouter, Together AI, Perplexity, Nebius Token Factory and Nvidia's NIM endpoint at build.nvidia.com, and through the NeMo framework for self-hosting. The license is OpenMDW-1.1, permitting commercial use, modification and downstream training. On Blackwell the deployment runs native FP4 at 5.03 bits per element; on Hopper it falls back to W4A16 because Hopper lacks native FP4 tensor cores.

The headline throughput claim is at 8K input and 64K output on GB200 under NVFP4: 5.9× faster than GLM-5.1-754B, 4.8× faster than Kimi-K2.6-1T and 1.6× faster than Qwen-3.5-397B. The comparison runs Nemotron on TRT-LLM and competitors on vLLM, each on the framework it is tuned for and disclosed in the tech report. The 1.6× over Qwen 3.5 is the load-bearing number because Qwen 3.5 was the previous open-weights speed leader. Cost-per-task-completion runs roughly 30% lower than the closest comparable on SWE-Bench Verified and Terminal Bench because Nemotron emits fewer tokens per turn at the same reasoning effort.

"Open innovation is the foundation of AI progress. With Nemotron, we're transforming advanced AI into an open platform that gives developers the transparency and efficiency they need to build agentic systems at scale."— Jensen Huang, Nvidia CEO, Nemotron 3 family preview

Benchmark-wise, Nemotron 3 Ultra posts 71.9 on SWE-Bench Verified, 56.4 on Terminal Bench 2.1, 570 on IOI 2025 competitive programming, and 94.7 on RULER at 1M tokens. On AA-Omniscience the model scores 78.7, the highest non-hallucination score in Nvidia's comparison set. Terminal Bench 2.1 is the one place Nemotron trails: Kimi-K2.6 leads at 67.2. Pretraining ran across 20 trillion tokens in two phases (15T diversity, 5T quality) including a 173B-token GitHub code refresh with a September 30, 2025 cutoff. Post-training combined SFT on 50M examples, RL across 2M tasks in 55 environments, and a multi-teacher on-policy distillation step. Datasets and recipes accompany the release. The Ultra weights take Nvidia from a credible open-weights contender to a credible open-weights leader on the agent workloads this generation targets.

Google's Texas Meitner campus pairs a data center with 1 GW of dedicated wind, solar and storage in a power-first template

Data Center Knowledge · 2026-06-05 · Source ↗ ·

Google's Meitner site in West Texas is the first hyperscaler campus designed power-first rather than network-first. A 1 GW dedicated renewable generation portfolio (wind, solar, storage) is co-located with the data hall before grid interconnect. The campus targets 95% on-site renewable supply for the first AI workloads in 2027, with no Texas ERCOT capacity reservation required.

Data Center Knowledge's June 5 reporting put Google's Texas Meitner campus on the record as the first hyperscaler data-center site designed power-first rather than network-first. Located in Hartley County in the Texas Panhandle, the campus pairs a 600-megawatt initial data-hall build with a co-located 1-gigawatt dedicated renewable generation portfolio: 400 MW of wind, 350 MW of solar PV, and 250 MW / 1,000 MWh of battery storage, all on Google-owned land with direct point-to-point connections to the data hall. The campus targets 95% on-site renewable supply for the first AI workloads in 2027 and avoids the Texas ERCOT capacity reservation queue that has held up other Texas data-center projects for two to three years.

The power-first model inverts the hyperscaler site-selection logic of the previous decade. Network proximity and water availability used to drive site selection; power availability was a constraint to be solved. Meitner makes power generation the primary design constraint and treats network connectivity as the secondary problem to be solved through dedicated long-haul fiber. The template matters for what other hyperscalers do next. Microsoft's announced Wisconsin site is reported to be following a similar power-first model on a SMR-anchored design. Meta's Hyperion Louisiana site has the same shape with a natural-gas anchor. Amazon's Indiana campus uses an SMR pilot. The week's other relevant data point: Alphabet raised $85 billion in fresh equity with a $10 billion Berkshire Hathaway anchor, with capex now projected at $190 billion through 2026, a number that pencils out only if a meaningful share of new build is power-first rather than grid-dependent.

"We designed Meitner to take electrons off our own generation portfolio before we touch the ERCOT grid. Every other site we built started from the grid and worked outward."— Google Cloud infrastructure VP, on the Meitner reveal call

The water story sits underneath the power story. Hartley County's hydrology is constrained, but the campus design uses an air-cooled architecture with a heat-pump-assisted dry cooling loop, eliminating the water-consumption profile of conventional evaporative cooling. The first phase consumes roughly 9 million gallons of water per year, a number Google says is below the per-capita household water consumption of a town the size of Stratford, the nearest population center. The campus has booked a 30-year ground-water rights allocation through the Texas Panhandle Groundwater Conservation District.

Trump signs the AI executive order: voluntary 30-day pre-release review, $42 billion in broadband funding tied to state-law repeal

White House / CNBC / The Register · 2026-06-02 · Source ↗ ·

The June 2 executive order replaces the killed 90-day mandatory pre-release framework with a 30-day voluntary review, ties $42 billion in BEAD broadband funding to state-law repeal of any law that could "burden" AI development, and stops short of the licensing regime Anthropic, Microsoft and OpenAI had collectively requested. The federal preemption fight Leading the Future is funding now has a federal lever.

President Trump signed the long-anticipated frontier-model executive order on June 2, three weeks after the 90-day mandatory pre-release framework was killed under pressure from David Sacks, Mark Zuckerberg and Elon Musk. The replacement is materially weaker than what the labs had been preparing for.
