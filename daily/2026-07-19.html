The catalyst was concrete. Kimi K3 opened at number one on Arena's Frontend Code leaderboard at Elo 1,679 against Anthropic Claude Fable 5 at 1,631 and OpenAI GPT-5.6 Sol at 1,618. Simon Willison flagged the compounding reduction in his same-day writeup.

"Beginning July 20, Claude Fable 5 will be included in all Max and Team Premium plans, at 50% of limits. Pro and Team Standard users will continue to have access to Fable via usage credits, and will receive a one-time $100 credit."— Anthropic official announcement

Willison attributed the reversal directly to competitive pressure. OpenAI's GPT-5.6 Sol had crossed a price-performance threshold in early July that undercut Fable 5 economics. Moonshot's Kimi K3 release on Thursday made the position acute. The two-tier lock-in

The Friday reversal locks in a permanent two-tier structure: Max and Team Premium as the Fable-included tier at $100 and $200 per month, Pro and Team Standard as the mid-tier without frontier-model subscription access. "I suspect Anthropic have realized that Fable is worth being in the subscription plans even if they have to dial back their own training-related use to make it happen."— Simon Willison, blog post

The move matters for agent workloads specifically. Fable 5 is Anthropic's most capable long-horizon model for the tool-use, computer-use, and multi-step planning workloads that dominate Claude Code and Claude Cowork usage. Permanent inclusion at 50 percent of Max limits materially changes the economics for coding-agent shops that had begun designing Fable 5 out of default configurations to avoid credit burn. ggml-org GitHub · 2026-07-18 · Source ↗

PR #25823 modifies src/models/dflash.cpp so quantized K/V cache reuse across DFlash steps preserves attention math. Ships 27 binaries covering macOS Arm64/x64, Linux CPU/Vulkan/ROCm 7.2/OpenVINO/SYCL, Windows CPU/CUDA 12/13/Vulkan/OpenVINO/SYCL/HIP, plus iOS XCFramework, Android arm64, and web UI.

ggml-org tagged llama.cpp build b10068 at 13:36 UTC Friday, a single-focus release implementing pull request #25823, "model: rotate injected K/V cache for DFlash." The change modifies `src/models/dflash.cpp` so that when DFlash speculative decoding runs against a K/V cache that has been quantized (a common configuration for keeping large-model inference within VRAM budgets), the injected key-value cache slots are correctly rotated to preserve the attention math. Without the rotation, quantized K/V cache reuse across speculative decoding steps produces attention calculation errors that degrade generation quality, particularly at longer contexts.

"model: rotate injected K/V cache for DFlash (#25823)"— llama.cpp b10068 release title

The release ships 27 binary assets covering macOS Arm64 and x64, Linux CPU, Vulkan, ROCm 7.2, OpenVINO, and SYCL, Windows CPU, CUDA 12 and 13, Vulkan, OpenVINO, SYCL, and HIP, plus the iOS XCFramework, Android arm64, and the web UI. DFlash is one of several fast speculative-decoding variants that llama.cpp has been folding in over the last month alongside TurboQuant and MTP, and the K/V quantization interaction is one of the harder correctness surfaces in that family because rotation is otherwise a no-op when cache slots are stored in full precision.

Downstream forks and impact

Community engagement was thin: the release shows a single thumbs-up and celebration reaction, reflecting that this is a targeted correctness fix rather than a headline feature. Downstream forks pick up the change quickly. beellama.cpp, which chases DFlash plus TurboQuant for a claimed 3x generation speedup and 7.5x more K/V cache in the same VRAM footprint on Strix Halo class hardware, will rebase on it. The DGX Spark GB10-optimized fork will follow. For end users running long-context inference on quantized caches, the fix is the difference between coherent multi-turn output and subtle attention drift that only surfaces at 32K-plus tokens.

"Rotate injected K/V cache when using K/V quantization."— PR #25823 description
