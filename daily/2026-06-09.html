The on-device tier is what Apple controls end-to-end; the cloud tier is what Apple pays Google roughly $1 billion a year to run.

Apple ships four named Foundation Models split across on-device Neural Engine and Private Cloud Compute

Apple's WWDC 2026 model lineup names four variants: AFM Core (on-device base), AFM Core Advanced (on-device multimodal sparse), AFM Cloud (Private Cloud Compute base, with a dedicated image generation variant), and AFM Cloud Pro (the heaviest tier, running on Google Cloud servers with NVIDIA GPUs under Apple's attestation protocol). Apple confirmed at the Platforms State of the Union that the models were trained with distillation help from Google's Gemini teachers but contain no Gemini code or infrastructure in production. The on-device tier handles routing, privacy and lightweight tasks; the cloud tier handles the agentic and heavy-compute requests.

AppleInsider's Monday-afternoon analysis by William Gallagher confirmed what the company had been signaling through the spring: the new Apple Foundation Models are "all-Apple through-and-through" despite the Gemini-collaboration headlines that dominated weekend press. The four-model lineup is structured around request difficulty and where the model runs.

AFM Core is the on-device base model, the successor to the 3B-parameter Apple Intelligence model the Foundation Models framework exposed to developers at WWDC 2025. It handles Writing Tools, summaries, Smart Reply, light Visual Intelligence and the requests that complete within Apple Neural Engine budget. AFM Core Advanced is a natively multimodal version with a sparse architecture; Apple's language for "sparse" indicates a mixture-of-experts variant where only a fraction of parameters activate per token, letting more capable behavior fit on-device without the latency or memory cost of a dense model. Apple did not disclose total parameter counts.

AFM Cloud is the Private Cloud Compute base model, the cloud tier the company uses for requests that exceed on-device capability. A dedicated AFM Cloud variant handles image generation and editing. AFM Cloud Pro is the heaviest model and the news inside the news: Apple is running it on Google Cloud servers using NVIDIA GPUs, wrapped in the Private Cloud Compute attestation protocol so external auditors can verify Apple's claim that Google does not see user traffic. This is the first time Apple has publicly named another hyperscaler as part of its inference path. The Gemini-trained-but-not-Gemini distinction matters because the legal and reputational stakes around Apple's model-training data overlap with separate questions about where Apple runs inference. Apple's statement: "We use none of the models that Google deploys to their customers, nor do we use the infrastructure and means by which they deploy models to their customers."

"AFM Cloud Pro handles agentic tools and most demanding tasks, deployed on Google cloud servers and NVIDIA GPUs while maintaining Private Cloud Compute certification." — Apple, Platforms State of the Union, June 8

How Gemini contributed

The Gemini role was distillation and training assistance. Apple's pretraining pipeline used Gemini-family teacher models to produce supervision targets for Apple's own student models, a technique that lets the smaller student inherit some of the teacher's capability without inheriting its parameters. The same pattern Meta described for Llama 4 codistilling from Behemoth. Apple's distinction is that the Gemini teachers were "loaned" through Google's Gemini Cloud APIs for the training run rather than fine-tuned or modified in place, and the resulting Apple weights are then deployed independently. AppleInsider's framing: "Apple has confirmed the end result is pure Apple technology and code." The price tag, reportedly roughly $1 billion a year, covers the Gemini access during training plus the ongoing Google Cloud and NVIDIA GPU capacity for AFM Cloud Pro inference.

The architectural choice has competitive consequences. By keeping the on-device tier under exclusive Apple control while paying Google for the frontier cloud capacity, Apple insulates the day-to-day Siri experience from frontier-model competitive shifts while still routing the hardest queries to whichever frontier model the user has selected through the Extensions framework. The bet is that the on-device tier is what Apple's hardware moat actually monetizes, and that the cloud tier is increasingly a commodity Apple should buy rather than build.

Xcode 27 is the largest single-version change to Apple's developer IDE since the SwiftUI transition. The framing is that local AI handles the latency-sensitive editing path while frontier models handle the heavier judgment calls.

The agent surface lives in the editor pane with full tab and split-window support. Each tab can run a different agent against a different scope of the codebase. Apple positioned the agents as task-completion tools rather than auto-complete extensions, picking up the language Cognition and Anthropic have used for Devin and Claude Code.

Dual-engine local plus cloud

The architecture matters because Apple is the first major IDE vendor to ship two separate inference paths in a single product. The local model is a highly tuned Apple Silicon variant — likely a sibling of AFM Core or AFM Core Advanced — that runs natively on the Neural Engine and produces code suggestions, documentation snippets and Swift-aware completions in real time, with the project structure and SDK surface as in-memory context. Heavy queries (multi-file refactors, full test suite generation, structural bug-finding across thousands of lines) hand off to a third-party model the developer has selected: Anthropic's Claude, OpenAI's GPT family or Google's Gemini coding model. Teams that prefer their own provider can configure Xcode 27 to route suggestions to a custom LLM endpoint instead.

"With new intelligence frameworks and agentic coding in Xcode 27, developers have the tools they need to focus on what they do best." — Susan Prescott, Apple VP of Worldwide Developer Relations

Core AI exposes the Neural Engine through a Swift API that handles model loading, tokenization, KV-cache management and on-device tool calling without forcing the developer to write low-level Metal kernels. The Foundation Models framework, which last year exposed only the 3B-parameter on-device model, now adds image input alongside text, custom skills (developer-defined tool plugins) and the ability to dispatch jobs to Private Cloud Compute when on-device budget is exceeded.

The pricing footnote landed loudly with the developer audience: members of the App Store Small Business Program with fewer than 2 million users get Private Cloud Compute access free. That gives small studios access to the same inference path Apple uses for AFM Cloud Pro without paying per-token rates. Siri AI not coming to EU or China at launch: Federighi blames Digital Markets Act access demands

MacRumors · 2026-06-08 · Source ↗ ·

Apple confirmed Monday that the enhanced Siri AI announced at WWDC 2026 will not ship in the European Union or China when iOS 27 and iPadOS 27 release later this year. Craig Federighi told MacRumors the European Commission demanded that any AI assistant in the EU have "nearly unlimited access to a user's device" with autonomous action capabilities including reading and sending messages, making purchases and accessing files across apps. Apple proposed a Trusted System Agent intermediary and an 18-month phased rollout; the Commission rejected the proposals. Siri AI remains available on macOS 27, visionOS 27 and watchOS 27 in the EU; iPhone and iPad features that depend on agentic device access do not ship.

MacRumors broke the EU and China carve-out under embargo Monday morning alongside the WWDC keynote. Apple's stance is that the European Commission's interpretation of Digital Markets Act interoperability requirements would force any AI assistant operating on an EU-distributed iPhone to expose autonomous capability — reading and sending messages, making purchases, accessing files across apps — to third-party assistants on equal terms. Apple's privacy and security framing is that this scope of access amounts to "nearly unlimited" device control and cannot be granted to unvetted third parties without breaking the device security model.

Apple proposed two mitigations. The first was a "Trusted System Agent" intermediary that would sit between any virtual assistant and the device's privileged surfaces, brokering access through a privacy-preserving permission layer that all assistants would share equally. The second was an 18-month phased rollout that would have given Apple and EU developers time to build against the intermediary before Siri AI launched in the region. Per Apple, the European Commission rejected both proposals without offering an alternative path that would let Apple ship Siri AI in the EU on the company's preferred privacy footing.

"We're deeply disappointed that our EU users won't have Siri AI. Their refusal to engage constructively on solutions that preserve privacy and security means we do not currently have a timeline." — Craig Federighi, Apple SVP of Software Engineering

What ships and what does not

The carve-out is partial. Siri AI does ship on macOS 27, visionOS 27 and watchOS 27 in the EU. The iPhone and iPad features that depend on the agentic device-access surfaces — reading messages, sending messages, navigating across apps, the new Siri conversation app, the expanded Visual Intelligence, the Camera-app Siri mode — do not. EU developers cannot test or integrate the iPhone-side Siri AI surfaces in their own apps either, which compounds the regional gap because EU-based developers building for global distribution lose visibility into how their apps behave on the new Siri stack.

The China track is separate but produced the same result. Apple has not publicly described the Chinese regulatory requirements that paused the Siri AI launch in the People's Republic, but the timing aligns with ongoing approval cycles for foreign AI services under the Cyberspace Administration of China. macOS and visionOS remain unaffected.

The political timing is awkward. The EU AI Act's high-risk obligations take effect August 2, 2026. Apple's carve-out lands six weeks before that deadline and reframes the EU-US frontier-AI policy gap from theoretical to immediate. EU users will see iPhones in stores that have the new Siri AI marketing collateral but a notice that the assistant is not available in their region. The Cupertino-Brussels relationship is the largest visible US-EU frontier-AI policy fracture to date.

App launch up 30 percent: the driver is a more aggressive prelink cache during cold app launch. The dynamic linker on iOS resolves library symbols when an app's binary loads; Apple's prelink cache stores precomputed link tables so the linker does less work on subsequent launches. iOS 27 expands the prelink cache to cover more of the system-library surface, so cold launches of apps that depend on UIKit, SwiftUI, Combine and the larger system frameworks have less work to do before the app's main thread runs.

Photos load 70 percent faster: the driver is the new Spotlight index plus the file-preloading layer that uses the index to predict which assets an app will request and warm them into memory before the request lands.
